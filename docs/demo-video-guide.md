# 演示视频录制指南

本文档用于录制 WS63E 环境巡检小车项目的最终演示视频。目标是用 3 到 5 分钟讲清楚完整闭环：

```text
小车采集环境数据 -> 星闪基站接收 -> 数据上云 -> App/Web 查看 -> App 控制小车 -> 巡检任务执行 -> 云端留痕
```

## 展示口径

推荐主线：

```text
我用手机通过基站控制小车，小车实时采集温湿度和光照，基站把数据上传到云端，云端记录设备状态、传感器数据、控制命令和巡检任务，手机和网页都能查看这些结果。
```

需要保持诚实的边界：

- 当前巡检路线提供两套模板：闭合环线巡检、折返通道巡检。
- 当前巡检不是地图导航或自动路径规划，而是 App 下发固件路线命令，由小车固件连续执行定时动作序列。
- 云端负责设备状态、环境数据、控制命令、巡检任务和分析结果的记录与展示。
- 不要展示或口播真实云端密钥。
- 不要声称多小车系统已经完成；可以说系统模型预留了后续扩展空间。

## 拍摄前准备

硬件：

- WS63E 巡检小车已上电。
- BearPi / WS63E 星闪基站已上电。
- 手机已安装最新 APK。
- 电脑打开 Web 页面或云端 API 数据页。
- 小车、手机、电脑屏幕尽量在同一桌面范围内。

软件和连接：

- 手机连接到正确 Wi-Fi。
- App 能切换到“基站”模式并收到环境数据。
- 小车可以通过 App 摇杆前进、停止、左转、右转。
- 巡检 tab 选择路线模板后点“一键下发任务”，小车能执行对应动作序列。
- 云端页面能看到设备、基站、环境 readings、recent commands 或 recent tasks。

建议提前准备的画面：

- 串口日志中出现 `DHCP success ip=...`。
- 串口日志中出现 `UDP bridge ready`。
- 串口日志中出现 `rx telemetry` 或同等遥测接收日志。
- 如果固件直连云已经修好，准备 `cloud_uplink uploaded telemetry` 成功日志。

## 推荐视频结构

### 1. 系统全景，10 到 15 秒

画面：

- 桌面全景。
- 小车、基站、手机 App、电脑 Web/云端页面同时入镜。

旁白：

```text
这是基于 WS63E 和星闪基站的环境巡检小车系统。小车通过星闪连接基站，基站负责数据汇聚和上云，手机端负责遥控和巡检操作，Web 端用于数据展示和管理。
```

### 2. 硬件联网与基站状态，20 到 30 秒

画面：

- 拍串口或终端成功日志。
- 拍基站在线状态。

重点日志：

```text
DHCP success ip=...
UDP bridge ready
rx telemetry
```

如果云端 HTTP 路径已经修好，可补充：

```text
cloud_uplink uploaded telemetry
```

旁白：

```text
基站已经接入局域网，并通过星闪接收小车环境数据。后续数据会进入平台，用于手机端和 Web 端展示。
```

### 3. 真实环境数据变化，30 到 40 秒

画面：

- 手机 App 或 Web 数据面板。
- 小车传感器在画面里。

推荐动作：

- 用手遮住光照传感器，展示光照变化。
- 手靠近传感器，展示温度或湿度趋势变化。
- 小车移动后，展示方向或状态字段变化。

旁白：

```text
小车实时采集温度、湿度和光照数据。这里遮挡光照传感器后，光照值会随真实环境变化更新。
```

### 4. App 基站模式遥控，40 到 60 秒

画面：

- 手机 App 和小车同时入镜。
- 手指操作摇杆，小车同步运动。

动作顺序：

```text
前进 -> 停止 -> 左转 -> 右转 -> 停止
```

注意：

- 每个动作控制在 1 到 2 秒。
- 不要长时间连续遥控，避免无线链路波动影响视频。
- 小车附近留足空间，避免撞到桌边或线缆。

旁白：

```text
手机端通过基站模式发送控制指令，基站再通过星闪链路转发给小车。这里可以看到 App 操作和小车运动是同步的。
```

### 5. 巡检 tab 和两套巡检线路，40 到 60 秒

画面：

- 手机进入“巡检”tab。
- 点击“一键下发任务”。
- 拍到路线模板里有“闭合环线巡检”和“折返通道巡检”。
- 拍到任务队列出现所选路线名称。
- 拍到时间线状态变化。
- 同时或切镜头拍小车执行所选路线动作。

推荐旁白：

```text
巡检任务提供闭合环线和折返通道两套模板。App 会记录任务创建、固件路线命令下发、巡检路线执行和完成状态。
```

注意：

- 不要说“自动路径规划”。
- 不要说“任意路线下发”。
- 可以说“当前提供两套可选巡检模板，云端任务模型保留了多步骤路线扩展能力”。

### 6. 云端数据和管理页面，30 到 50 秒

画面：

- Web 或 App 云端模式。
- 展示设备在线、基站在线、环境 readings、recent commands、recent tasks、Agent 分析报告。

旁白：

```text
云端保存设备状态、环境数据、控制命令和巡检任务记录。手机端和 Web 端查看的是同一套平台数据，用于赛后追溯和状态分析。
```

如果正式云端仍存在 HTTP/HTTPS 配置问题：

```text
这里展示的是云端 API 接入后的平台数据效果。板端直连云的 HTTP 入口需要服务器侧放行 /ws63-api 路径，避免被重定向到 HTTPS。
```

### 7. 收尾总结，15 到 20 秒

画面：

- 手机、Web 页面、小车同时入镜。

旁白：

```text
系统完成了小车环境采集、星闪基站转发、云端数据管理、移动端遥控和巡检任务记录的完整闭环。
```

## 实际拍摄顺序

为了降低翻车风险，建议按稳定性拍，不按最终剪辑顺序拍：

```text
1. 硬件全景
2. 基站模式遥控
3. 环境数据变化
4. 巡检任务
5. 云端页面
6. 串口和日志补拍
```

如果某一段失败，可以重拍该段，不影响其他素材。

## 必拍清单

- [ ] 小车、基站、手机、电脑同框。
- [ ] App 基站模式收到真实环境数据。
- [ ] 光照或温湿度有一次明显真实变化。
- [ ] App 摇杆控制小车运动。
- [ ] App 停止指令能让小车停下。
- [ ] 巡检 tab 可选择闭合环线或折返通道。
- [ ] 小车执行所选巡检路线动作。
- [ ] 巡检任务队列或时间线显示真实任务状态。
- [ ] 云端或 Web 页面显示设备/基站/数据/命令/任务记录。

## 避坑清单

- 不要展示真实 `DEVICE_INGEST_KEY`。
- 不要长时间连续遥控。
- 不要把失败日志、报错弹窗、断连画面放进最终视频。
- 不要让手机和电脑屏幕过曝。
- 不要在画面里出现无关聊天记录、密钥、私人路径或账号信息。
- 不要声称小车支持地图导航、路径规划、自动避障或多车实车调度，除非当时已经真实完成并录到证据。

## 备用素材建议

如果实车临时不稳定，至少保留这些备选镜头：

- App 数据页显示历史曲线。
- Web dashboard 显示 readings 和 recent commands。
- PowerShell smoke test 成功输出。
- 串口中基站 DHCP、UDP bridge ready、rx telemetry 日志。
- PC bridge 上传成功日志。

这些素材不能完全替代实车演示，但可以证明平台链路和数据模型已经打通。
